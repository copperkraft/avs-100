Описание интерфейса симулятора: В верхней части окна расположены три кнопки: — "run" — запускает выполнение вашей программы; — "clear" — очищает поле от введенных вами команд; — "step" — выполняет один шаг из вашей программы (одну команду). Ниже располагается поле регистров и флагов. Это поле отражает содержимое шестнадцати доступных регистров и флага CF (необходим при работе с отрицательными числами). Справа от поля "Register and Flags" находится поле команд. Непосредственно туда и вводятся команды пользовательской программы. Две цифры перед командой — её порядковый номер. Для первой команды это "00". В правой части окна лежит поле заданий и результата. Оно состоит из нескольких частей: — текстовое описание задания; — столбец "in" — может содержать числа, которые стоят на входе симулятора. — столбец "need" — содержит числа, которые должны получиться на выходе в случае правильного выполнения задания. — столбец "out" — содержит числа, которые получились на выходе симулятора в результате выполнения программы пользователя. _______________________ Как это работает: Суть заданий в этом симуляторе сводится к тому, чтобы написать программу, которая при выполнении всех условий задания будет иметь на выходе числа из столбца "need". Команды вводятся в поле команд через "Enter". Общий синтаксис команды: КОМАНДА ОПЕРАТОР Например, OUT 34 Некоторые команды могут содержать несколько операторов: КОМАНДА ОПЕРАТОР1 ОПЕРАТОР2 Например, ADD R0 R1 Особенность этого симулятора в том, что команды выполняются "по кругу": т.е. после выполнения последней команды снова выполняется первая. И так до тех пор, пока вы не нажмёте кнопку "stop". Также программа остановится автоматически в случае правильного выполнения задания, когда весь столбец "out" заполнится нужными числами. Регистры: В курсе АВС будет дано чёткое понятие регистров, но для решения задач в этом симуляторе достаточно воспринимать их как ячейки (контейнеры, если хотите), в которых могут содержаться числа. Посмотреть содержимое регистров можно в поле "Register and Flags". По умолчанию они пусты, что отображается как, например, "R0: 0". В этом поле "R5 12" обозначает, что в регистре с номером 5 находится число 12. Запись в регистры может осуществляться разными путями. Например, самый простой: с помощью команды INP. Она имеет следующий синтаксис: INP РЕГИСТР Например, INP R6 Эта команда записывает число со входа симулятора в регистр, указанный в команде. Что такое вход (столбец "in") в данном случае? Это как бы "обойма", в которую по условию задания записаны числа. Команда INP забирает верхнее число, а весь столбец поднимается на одну ступень вверх. То есть второе число становится первым, третье — вторым и так далее. Например, в столбце "in" записаны числа 1, 12, 43, тогда в результате выполнения кода: INP R0 INP R10 INP R1 в регистр 0 будет записано число 1, в регистр 10 — 12, а в регистр с номером один попадёт число 43. Также запись может осуществлять другими командами. Например, командой сложения ADD. Её синтаксис: ADD РЕГИСТР1 РЕГИСТР2 Эта команда выполняет операцию сложения содержимого двух регистров и записывает результат в регстр, который следует за командой ADD (в данном случае РЕГИСТР1). Допустим, в R0 у нас содержится число 2, а в R3 — 5. Тогда команда ADD R0 R3 запишет в R0 число 7. Эта же команда позволяет добавлять числа к содержимого регистров. Например, команда ADD R3 12 "приплюсует" к числу в R3 12 и запишет это в R3. Вывести число из регистра можно командой OUT. Например, в R4 содержится число 2. Тогда команда OUT R4 отправит на выход симулятора число 2.
